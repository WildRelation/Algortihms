# F7 — Träd och binära sökträd

**Ämnen:** Träd, Binära träd, BST, BST i JCF, Implementering av BST

## Kärninnehåll

**Träd-terminologi:**
- Rotnod, nod, löv, förälder, barn, nivå, höjd
- N noder → N-1 bågar
- Träd = graf där det finns exakt en väg mellan varje par noder

**Binärt träd** — varje nod max 2 barn
- **Full:** varje nod har 0 eller 2 barn
- **Perfekt:** alla löv på samma nivå (alla nivåer fulla)
- **Komplett:** alla nivåer fulla utom sista, vänsterjusterad
- **Balanserat:** höjdskillnad max 1 för vänster/höger subträd per nod

**BST (Binärt sökträd):**
- Vänster subträd < rotnoden < höger subträd
- Unika nycklar
- Sökning: O(log n) om balanserat, O(n) i värsta fall (länkad lista)
- Insättning: O(log n) att hitta plats, O(1) att hänga på löv

**Jämförelse med andra strukturer:**
| Struktur | Sökning | Insättning/Borttagning |
|----------|---------|------------------------|
| Sorterad array | O(log n) | O(n) |
| Länkad lista | O(n) | O(1) |
| Balanserat BST | O(log n) | O(log n) |

**Traversering (rekursiv):**
- **Inorder:** vänster, nod, höger → sorterad ordning (vanligast)
- **Preorder:** nod, vänster, höger
- **Postorder:** vänster, höger, nod

**Ta bort nod — 3 fall:**
1. Löv: sätt förälderns pekare till null
2. Ett barn: ersätt noden med barnet
3. Två barn: ersätt med minsta nod i höger subträd (den noden har inget vänsterbarn)

**BST i JCF:** `TreeMap` och `TreeSet` — Red-Black Tree (självbalanserande BST, garanterar O(log n))

**Självbalanserande BST:** Krävs om data sätts in i sorterad ordning (annars degenererar till länkad lista).
