# F3 — Stack

## ADT Stack — LIFO (Last In First Out)
En stack fungerar som en hög: man lägger på och tar bort från toppen.
Processorn använder en stack för att hålla reda på returadresser vid funktionsanrop.

**Måste-metoder:** `push(element)`, `pop()`
**Vanliga:** `empty()`, `peek()`

## Stack i JCF

**`java.util.Stack<E>`** — ärver från `Vector<E>` (ärver felaktiga metoder, synkroniserad, rekommenderas EJ)

**`Deque<E>`** — interface, implementeras av `LinkedList` och `ArrayDeque`
- Har push, pop, peek + isEmpty
- Ej null-element tillåtna
- Kursens rekommendation: `Deque<Integer> stack = new ArrayDeque<>();`

## Tillämpning: Omvänd polsk notation (postfix)
Operanden skrivs efter operatorerna: `12 6 + 3 /` = (12+6)/3 = 6

**Algoritm:**
```
skapa tom stack
för varje token:
  om siffra: push(token)
  om operator:
    höger = pop()
    vänster = pop()
    push(vänster OP höger)
returnera pop()
```

**Körexempel `12 6 + 3 /`:**
- push 12 → [12]
- push 6 → [6, 12]
- `+` → pop 6, pop 12, push 18 → [18]
- push 3 → [3, 18]
- `/` → pop 3, pop 18, push 6 → [6]
- return 6 ✓

## Array-implementation (ArrayStack)
**Nyckelidé:** Toppen = sista elementet (index `top`). top startar på -1.

```java
public E push(E obj) {
    if (top == maxSize-1) reallocate();
    data[++top] = obj; return obj;
}
public E pop() {
    if (empty()) throw new EmptyStackException();
    return data[top--];
}
public E peek() { return data[top]; }
public boolean empty() { return top == -1; }
```
**Alla metoder O(1)**. Amorterat O(1) med reallocate (dubbling).

## Länkad lista-implementation (LinkedStack)
Enkellänkad lista. Toppen = head-noden.

```java
public E push(E obj) { top = new Node<>(obj, top); return obj; }
public E pop() { E r=top.data; top=top.next; return r; }
public E peek() { return top.data; }
public boolean empty() { return top == null; }
```
**Alla metoder O(1)**. Exakt rätt storlek alltid.

## Minnesjämförelse
| Impl. | Minnesbehov | Förändras? |
|-------|------------|-----------|
| Array | Max 4× nödvändigt (om krymper) | Ja |
| Länkad lista | Alltid 2× nödvändigt (pekare) | Nej |
→ Länkad lista mer förutsägbar. Array snabbare pga cache-lokalitet.
